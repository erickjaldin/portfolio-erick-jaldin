# Batalha de Modelos & Engenharia de Prompt (XML) 🧠📊

Este projeto foi um estudo comparativo prático realizado para avaliar como diferentes modelos de Inteligência Artificial interpretam prompts estruturados em blocos XML.

🚀 **[Acessar Relatório Completo (Word)](https://d.docs.live.net/282A8B38D2D509C1/Documentos/planilha-teste-IA.docx)**

---

## 🎯 O Objetivo do Prompt
O desafio consistia em rodar um prompt estruturado em XML para gerar uma página HTML Single Page responsiva e minimalista sobre o tema **"Tipos de Pneu Ideal"**, contendo menu de âncoras, galeria com imagens, tamanhos e marcas de pneus.

---

## 📈 Resultados da Análise Comparativa

O prompt foi testado em sete ferramentas diferentes. Abaixo estão os destaques das performances reais observadas:

* **Maior Compreensão:** O **Grok** se destacou ao seguir rigorosamente as diretrizes de design, entregando o conteúdo solicitado com um menu fluido e muito bem otimizado.
* **Eficiência de Tokens:** O **Qwen** foi o mais direto e econômico, gerando o código com apenas **700 tokens**. O Grok e o Claude foram os mais verbosos, passando de 4000 tokens para o mesmo resultado.
* **Falhas Identificadas:** O GPT errou no carregamento das imagens, o Qwen esqueceu do menu de navegação e o Gemini não reconheceu os parâmetros do prompt neste teste.

---

## 🛠️ Conclusão Prática

* **Para Prototipagem Rápida:** **Qwen**, devido à sua entrega direta, leve e baixo consumo de tokens.
* **Para Códigos Complexos:** **Grok**, pela capacidade superior de entregar uma estrutura de menu otimizada, fluida e com excelente interpretação do contexto visual exigido.
